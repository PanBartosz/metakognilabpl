# Metakognilab.pl Design Principles

## Source of truth

- Treat the original WordPress site at `meta.kognilab.pl` as the primary source for public-facing copy.
- Stay close to the original wording, especially on the homepage. Edit lightly for clarity, length, and layout only.
- Do not invent marketing copy, explanatory filler, or generic research-site language when the source content already exists.
- Never mention migration, Astro, WordPress, rebuilds, archives, or implementation details anywhere on the public site.

## Visual direction

- The project logo is the main visual reference. Derive form, rhythm, and motifs from it.
- Use a cool slate and cold off-white palette. Avoid warm beige, cream, or yellowish paper tones.
- Prefer hard edges and clean rectangular framing over rounded, soft, generic UI.
- Avoid typical AI-generated design habits: glassy cards, oversized rounded corners, safe stock-layout aesthetics, and decorative visuals without a reason.
- Use simple geometric shapes and patterns inspired by the logo instead of unrelated illustrations.
- Keep only the project logo and real team photos as image assets unless a future request explicitly adds something else.

## Layout and composition

- Alignment, spacing, whitespace, and flow are critical. Treat them as first-order design decisions.
- Build compositions that feel intentional and calm. Do not let elements merely sit near each other; they need a shared alignment system.
- Avoid clever card mosaics when they make the layout feel forced. Prefer simpler, more regular grids.
- Do not stretch short-content cards just to equalize heights. Empty space is better than broken alignment or awkward vertical distribution.
- If a section does not clearly add information, remove it instead of decorating it.

## Content discipline

- Avoid duplication. Do not repeat the same information in the header, hero, cards, labels, and supporting text.
- Do not restate headings in weaker prose directly underneath them.
- Do not add obvious meta-copy such as “this section will list...” or similar filler.
- For missing content, use the smallest possible placeholder or omit the section entirely.
- Placeholder sections should be visually quiet and text-light.

## Header

- The current header is approved and should be treated as stable unless the user explicitly asks to revisit it.
- Keep the header clean. No poster links, emails, utility buttons, or secondary promotional items in the masthead.
- The header should contain only the brand lockup, affiliation, funding, and navigation.

## Homepage structure

- Keep the homepage organized as four main content panels beneath the header:
- `About the project`
- `Latest papers and conference talks`
- `Current materials`
- `Blog entries`
- `About the project` should stay closest to the original WordPress homepage copy.
- `Latest papers and conference talks` should remain minimal until real items are provided.
- `Current materials` should be a simple list with a follow-through link to resources.
- `Blog entries` should be a straightforward vertical list, not a promotional block.

## Practical review standard

- Before considering a page done, check for:
- repeated labels or headings
- placeholder prose that says nothing
- elements that are misaligned despite sharing a row or column
- oversized type that overpowers the page title or section hierarchy
- cards whose spacing looks stretched only to satisfy a grid
- any section whose function is decorative rather than informational
